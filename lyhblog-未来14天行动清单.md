# lyhblog 未来14天行动清单

更新时间：2026-04-17
适用目录：`D:\Monkey-blog`

## 总目标

这 14 天不再继续无节制加功能，重点做三件事：

- 项目收口
- 项目上线
- 项目简历化

你的当前主线应该是：

1. 把 `lyhblog` 做成一个能展示、能讲清楚、能跑通的成品
2. 把 `AI 写作助手` 稳定下来
3. 把项目包装成简历项目
4. 再开始第二个更贴近 Python 后端 / 爬虫方向的项目

---

## 第 1 周：把项目做成成品

### 第 1 天：修时间与基础配置

目标：把项目运行基础修正到正常状态。

任务：

- 把 Django `TIME_ZONE` 改成 `Asia/Shanghai`
- 保持 `USE_TZ = True`
- 重启后端，确认文章创建时间显示为中国时间
- 检查 `.env` 是否已经能稳定读取
- 检查 `ANTHROPIC_*` 配置是否都已经落地

验收：

- 新建文章时间显示正确
- AI 接口仍然可用

### 第 2 天：收口 AI 写作助手配置

目标：让 AI 功能不再靠临时环境变量跑。

任务：

- 整理后端 `.env`
- 整理后端 `.env.example`
- 整理前端 `.env.example`
- 确认模型名、超时、网关地址统一
- 把错误提示从技术报错改成用户可读提示

验收：

- 换一个新终端也能直接启动项目
- AI 助手可以成功返回建议

### 第 3 天：修文案和乱码

目标：让页面看起来像完整项目，而不是实验状态。

任务：

- 修中文乱码
- 修后台和前台按钮文案
- 修 AI 助手的提示语
- 修后端返回信息里的奇怪文本
- 检查分类、标签、错误提示是否自然

验收：

- 页面里主要中文文案可正常阅读
- AI 助手结果区和保存表单都能正常理解

### 第 4 天：走通 AI 助手完整闭环

目标：完整验证一条真实业务链。

任务：

- 后台登录管理员
- 打开新增文章页
- 点击 `AI 助手`
- 生成标题、摘要、标签、提纲、草稿
- 一键回填表单
- 点击保存文章
- 前台查看文章详情

验收：

- 这条链路至少完整跑通 1 次
- 截图保留结果

### 第 5 天：整理 README 第一版

目标：把仓库说明写成别人能看懂的版本。

README 至少包含：

- 项目介绍
- 技术栈
- 目录结构
- 本地启动方式
- AI 助手说明
- 环境变量说明
- 页面截图

验收：

- 新人拿到项目后，能按 README 复现启动

### 第 6 天：做项目截图和演示素材

目标：为简历和 GitHub 准备可展示内容。

建议截图：

- 首页
- 文章详情页
- 后台文章管理页
- 新增文章页
- AI 写作助手抽屉
- 登录页

如果有余力：

- 录一段 30 秒到 1 分钟的小演示

验收：

- 至少准备 4 张清晰截图

### 第 7 天：修部署文档

目标：让部署文档和当前代码一致。

任务：

- 按当前代码重写部署步骤
- 把 `.env` 方案写进去
- 把 AI 网关配置写进去
- 把前端构建步骤写进去
- 把 Ubuntu / WSL 跑法写进去

验收：

- 文档不再和仓库现状打架

---

## 第 2 周：把项目变成简历项目

### 第 8 天：WSL 或 Ubuntu 部署演练

目标：先在 Linux 环境里走一遍。

任务：

- 后端 `Gunicorn`
- 前端 `npm build`
- Nginx 反向代理
- MySQL 连接
- AI 网关配置

验收：

- 本地 Linux 环境能打开页面并调通接口

### 第 9 天：整理上线方案

目标：为家里 Ubuntu 或云服务器部署做准备。

任务：

- 梳理域名 / IP
- 梳理后端端口
- 梳理前端静态资源目录
- 梳理 MySQL 数据库配置
- 梳理 `.env.production`

验收：

- 你自己能清楚说出前后端怎么部署

### 第 10 天：写中文简历项目描述

目标：把项目从“个人博客”升级成“可投递项目”。

推荐名称：

`前后端分离内容管理系统 + AI 写作助手 | Django REST Framework + Vue3`

建议输出：

- 项目介绍 1 段
- 项目亮点 4 条
- 技术栈 1 行
- 遇到的技术问题 2 到 3 条

验收：

- 简历里能直接粘贴使用

### 第 11 天：写 GitHub 项目介绍

目标：让 GitHub 首页能直接体现项目价值。

任务：

- 仓库标题更清晰
- README 封面图
- 功能特性列表
- 技术栈徽章可选
- AI 助手说明单独成节

验收：

- 别人点开仓库首页就知道你做了什么

### 第 12 天：设计第二项目

目标：开始为 Python 后端 / 爬虫方向补第二个项目。

建议项目：

`新闻/文章数据采集与管理平台`

最小版范围：

- 爬虫采集文章
- 去重入库
- 后台查看采集结果
- 简单任务状态与失败重试

验收：

- 写出项目说明和功能范围，不急着写代码

### 第 13 天：拆第二项目最小需求

目标：避免第二项目一开始做太大。

拆成最小模块：

- 数据源采集
- 去重逻辑
- 存储逻辑
- 管理后台
- 结果展示
- 定时任务

验收：

- 输出一个简洁的需求清单

### 第 14 天：练口述项目

目标：你要能把项目讲明白。

至少练会这几件事：

- 项目做了什么
- 技术栈为什么这样选
- AI 助手是怎么接进去的
- 遇到过哪些坑
- 为什么现在不先上 LangChain / LangGraph
- 如果继续做，下一步准备怎么扩展

验收：

- 1 到 3 分钟项目介绍能顺畅说出来

## 这 14 天的优先级原则

### 必须优先做

- 项目收口
- AI 功能稳定
- README 和截图
- 部署演练
- 简历描述

- 继续堆新功能

- 没收口就跳到第二项目实现
