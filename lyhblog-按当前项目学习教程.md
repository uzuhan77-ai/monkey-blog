# lyhblog 按当前项目学习教程

> 目标：不是从零照抄原作者，而是基于你现在这个项目，边读边改，最后写出一个属于你自己的、风格简洁舒服的 blog。
> 适合你现在的状态：项目已经有骨架，但还没有完全打磨好。

---

## 一、你现在的项目到哪了

你现在不是空项目，已经有这些东西：

- 前台：首页、文章详情、登录、注册、评论区骨架
- 工程化：`request.js`、`router`、`Pinia user store`

1. 先看懂现有代码
2. 再修掉关键小坑
3. 再按自己的审美重写页面
4. 最后再补功能

## 三、推荐学习顺序

### 第 0 步：先把项目地图吃透

先认这几个目录：

- `lyhblog-frontend/src/views`：页面
- `lyhblog-frontend/src/components`：可复用组件
- `lyhblog-frontend/src/api`：接口请求
- `lyhblog-frontend/src/router`：页面跳转规则
- `lyhblog-frontend/src/stores`：登录状态

这一阶段只做一件事：

- 看到一个页面时，知道它的数据从哪来、点按钮后去哪、调用了哪个接口

---

### 第 1 步：先学首页列表

先读这几个文件：

- `lyhblog-frontend/src/views/Home.vue`
- `lyhblog-frontend/src/api/article.js`
- `lyhblog-frontend/src/utils/request.js`

你要看懂的不是语法，而是链路：

1. 页面加载时调用 `getArticleList`
2. `getArticleList` 去调 `ApiArticleList`
3. `ApiArticleList` 走 `request`
4. 后端返回数据后，渲染成卡片列表

你这一课要学会的点：

- `ref` 是怎么存列表数据的
- `onMounted` 为什么要在页面加载时请求
- 分类切换为什么要把页码重置为 1
- 为什么接口层要单独放到 `api/`

你这一课的练习任务：

- 自己把首页文章卡片改得更简洁一点
- 不改功能，只改排版、留白、字体大小、按钮文案

验收标准：

- 首页文章能正常显示
- 分类筛选正常
- 分页正常
- 你能讲清楚“首页数据流”怎么走

---

### 第 2 步：学文章详情和评论区

先读这几个文件：

- `lyhblog-frontend/src/views/ArticleDetail.vue`
- `lyhblog-frontend/src/components/CommentSection.vue`
- `lyhblog-frontend/src/api/comment.js`

你要看懂的核心：

1. 详情页怎么从路由里拿 `id`
2. 为什么评论区做成组件，而不是全塞进详情页
3. 为什么“是否登录”要从 store 里拿
4. 评论发布成功后为什么要重新加载评论列表

你这一课的练习任务：

- 把详情页做得更像“真正可阅读的文章页”
- 重点改：标题区、时间信息、正文宽度、评论区间距

验收标准：

- 进入文章详情能正常显示正文
- 评论列表能显示
- 登录后可以发表评论
- 你能讲清楚“页面组件”和“业务组件”的区别

---

### 第 3 步：学登录、状态管理、路由守卫

先读这几个文件：

- `lyhblog-frontend/src/views/Login.vue`
- `lyhblog-frontend/src/stores/user.js`
- `lyhblog-frontend/src/router/index.js`

这一块最重要，不难，但非常值钱。

你要看懂：

1. 登录成功后 token 存到哪
2. 为什么 store 里要有 `isLogin`
3. 为什么访问后台前要先过 `beforeEach`
4. `redirect` 参数为什么能让用户登录后跳回原页面

你这一课的练习任务：

- 自己把登录页改得更舒服，不要默认表单样子
- 简洁即可：一个卡片、一点留白、明确主按钮

验收标准：

- 未登录进 `/admin` 会跳到 `/login`
- 登录后能回跳
- 退出登录后权限失效

---

### 第 4 步：学后台骨架

先读这几个文件：

- `lyhblog-frontend/src/views/admin/Layout.vue`
- `lyhblog-frontend/src/views/admin/ArticleList.vue`
- `lyhblog-frontend/src/views/admin/ArticleEdit.vue`
- `lyhblog-frontend/src/views/admin/CommentList.vue`

这一阶段先不要追求“后台做很漂亮”，重点是学结构：

- 左侧菜单怎么切换页面
- 子路由怎么挂在 `/admin` 下
- 列表页、编辑页为什么拆开
- CRUD 页面为什么通常都有“列表 + 表单”两个层级

你这一课的练习任务：

- 先把后台页面结构理顺
- 再把文案和间距统一一下

验收标准：

- 你能说清楚后台每个页面负责什么
- 文章列表页、编辑页、评论页的职责不混乱

## 四、你这个项目当前值得优先修的小坑

这些不是让你现在一次性全修，而是我们后面可以拿来练手：

- 后台子路由里有几处 `component` 拼写写成了 `conponent`
- 评论管理页里变量名和生命周期调用有明显不一致
- 文章删除接口路径里有拼写问题
- 现在有些页面样式能用，但还不够统一

这类问题最适合拿来训练你：

- 怎么读报错
- 怎么定位文件
- 怎么做最小修复
- 怎么验证修没修好

## 五、风格建议：不要照抄原作者

你可以仿结构，但不要仿成一模一样。

你这个 blog 更适合走这条路线：

- 颜色少一点：白、灰、黑，加一个低饱和主色
- 留白多一点：卡片、正文、评论区都别太挤
- 字体层级清楚：标题明显大，正文安静，辅助信息更淡
- 首页别堆太多模块：文章列表 + 分类 + 简单顶部就够了

你只要做到这四点，页面就会自然显得“简洁舒服”。

## 六、最近 5 次学习建议

你接下来可以按这个节奏来：

1. 第一次：只学首页列表和接口链路
2. 第二次：只学文章详情和评论区
3. 第三次：只学登录、store、路由守卫
4. 第四次：只学后台布局和路由关系
5. 第五次：开始统一视觉风格，做成你自己的样子
