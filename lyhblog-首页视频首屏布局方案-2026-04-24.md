# lyhblog 首页视频首屏布局方案

日期: 2026-04-24

适用文件:
- `D:\Monkey-blog\lyhblog-frontend\src\views\Home.vue`

目标:
- 不做强轮播，不做旅游站那种很重的幻灯片
- 首页首屏改成一个简约的视频 Hero
- 整体气质参考 Claude，重点是留白、安静、内容感
- 保留你现在已有的文章搜索、分类、标签、文章列表逻辑

---

## 一、首页最终结构

首页建议只保留 3 个主要区块:

1. 视频首屏 Hero
2. 文章区
3. 项目说明区

这样结构足够清晰，也不会显得页面太空。

---

## 二、首屏 Hero 怎么搭

首屏不要做成左右信息特别多的面板，而是做成:

- 上方: `FrontHeader`
- 中间: 一块大的视频区域
- 视频上层: 文案、按钮、小状态信息
- 视频下方或右下角: 2 到 3 个信息卡

推荐布局如下:

```text
┌──────────────────────────────┐
│ FrontHeader                  │
├──────────────────────────────┤
│ Hero Section                 │
│  ┌────────────────────────┐  │
│  │ 左侧文案               │  │
│  │ 标题                   │  │
│  │ 副标题                 │  │
│  │ 按钮区                 │  │
│  │ 当前状态               │  │
│  │                        │  │
│  │              视频区域  │  │
│  │              覆盖柔和遮罩│ │
│  └────────────────────────┘  │
│   下方小卡片: 技术栈 / 当前重点│
└──────────────────────────────┘
```

### 首屏必须保留的元素

- 小标签: `LYHBLOG / PERSONAL CONTENT SYSTEM`
- 主标题
- 副标题
- 两个主要按钮
- 一个当前阶段说明
- 视频容器
- 2 到 3 个状态卡片

### 主标题建议

主标题不要太长，建议控制在 1 到 2 行。

可直接用这版:

`把 blog 做成一个真正可展示的项目`

### 副标题建议

`用 Django 与 Vue 搭建前后端分离的内容系统，记录学习、沉淀内容，也把它当成我的工程作品。`

### 按钮建议

- `查看文章`
- `查看项目`

第三个入口如果你想保留，可以做成弱按钮:

- `关于我`

### 当前阶段说明

可以直接写:

`当前阶段：先把博客首页、文章体验和内容结构打磨完整，AI 功能后置。`

---

## 三、视频区域怎么放

这里不是让你先做特效，而是先把结构留出来。

推荐写法:

- 外层一个 `hero-media`
- 里面一个 `video-shell`
- `video` 标签先预留
- 没有视频文件时，先放一个占位层

逻辑建议:

1. 先把布局写好
2. 视频文件后面再换
3. 如果暂时没有视频，就先显示占位文案

占位文案建议:

`这里后续替换为项目录屏视频`

### 视频素材建议

最适合你的不是风景视频，而是:

- 你博客首页录屏
- 后台文章管理录屏
- 页面滚动、搜索、文章切换的录屏

这样更像“项目展示”，不是单纯做视觉包装。

---

## 四、首屏下面的信息卡放什么

视频首屏下方最多放 3 张卡片，不要多。

建议内容:

1. 文章总数
2. 分类数量
3. 当前重点

如果你不想把“当前重点”放成数字卡，可以改成文字卡。

示例:

- `文章总数`
- `分类数量`
- `当前重点：博客体验 / 内容结构 / 后台完善`

如果你觉得数字卡太像后台仪表盘，那就换成这 3 个标签卡:

- `Django + DRF`
- `Vue3 + Element Plus`
- `内容管理 / 可展示 / 可部署`

---

## 五、文章区怎么接

Hero 下面不要再做很花的过渡，直接进入文章区。

文章区继续保留你现在已有的逻辑:

- 搜索框
- 分类筛选
- 标签筛选
- 文章列表
- 分页

但是结构建议改成下面这种顺序:

1. 区块标题
2. 一行简短说明
3. 筛选面板
4. 文章卡片列表
5. 分页

### 区块标题建议

- 小标题: `ARTICLE LIST`
- 主标题: `最新文章`

### 区块说明建议

`先把内容系统做完整，再继续打磨项目展示和部署能力。`

---

## 六、文章卡片怎么排

文章卡片不用重新发明，保留你现在的字段就够了:

- 分类
- 发布时间
- 标题
- 摘要
- 标签
- 查看详情提示

建议卡片顺序:

1. 顶部信息行
2. 标题
3. 摘要
4. 标签
5. 底部跳转提示

注意:
- 摘要建议做纯文本截断
- 如果 `content` 里有 HTML，先去标签再截断

---

## 七、项目说明区怎么写

文章区后面再加一个很轻的项目说明区，不要做成大而全的介绍页。

建议结构:

- 左侧: 一段项目说明
- 右侧: 3 个要点卡片

### 项目说明标题

`现在的重点不是扩功能，而是把博客体验做扎实`

### 项目说明正文

`这个项目当前聚焦首页、文章详情、分类标签和后台管理，先把它做成一套完整、可展示、可持续迭代的内容系统。`

### 右侧 3 个要点卡

建议写:

1. `首页重新定型`
   视频首屏、内容导向、简约风格

2. `文章体验优先`
   搜索、筛选、阅读承接、摘要清晰

3. `后台继续补齐`
   分类、标签、评论、管理入口整理

---

## 八、移动端怎么收

移动端不要照搬桌面布局。

移动端建议:

1. Header 保持简洁
2. 首屏改成上下结构
3. 视频区域放上面
4. 文案和按钮放下面
5. 信息卡纵向排列
6. 文章列表改成单列

移动端结构示意:

```text
Header
视频区域
标题
副标题
按钮
状态卡
文章筛选
文章列表
项目说明
```

## 九、你改 `Home.vue` 时建议怎么拆

建议把模板直接拆成下面这几个大块:

```vue
<template>
  <div class="home-page">
    <FrontHeader />

    <section class="hero-section">
      <div class="hero-copy"></div>
      <div class="hero-media"></div>
    </section>

    <section class="article-section"></section>

    <section class="project-section"></section>
  </div>
</template>
```

如果你想更贴近“视频主视觉”，也可以改成:

```vue
<template>
  <div class="home-page">
    <FrontHeader />

    <section class="hero-section">
      <div class="hero-video-panel">
        <div class="hero-overlay-content"></div>
      </div>
      <div class="hero-info-cards"></div>
    </section>

    <section class="article-section"></section>
    <section class="project-section"></section>
  </div>
</template>
```

这两种都可以。

如果你更想靠近 Claude 风格，我更推荐第二种。

## 十、脚本里保留哪些东西

你现在脚本里的这些逻辑建议原样保留:

- `getArticleList`
- `getCategoryList`
- `getTagList`
- `handleSearch`
- `handleCategoryClick`
- `handleTagClick`
- `handlePageChange`
- `goToDetail`
- `scrollArticles`

你只需要额外加一些用于首屏展示的数据，例如:

- `heroVideoSrc`
- `heroTechStack`
- `statusCards`
- `projectHighlights`

也就是说:
- 数据请求逻辑不要大改
- 主要改模板结构和展示文案

## 十一、建议你顺手一起修的点

你现在这个页面里有几个顺手就该修掉的问题:

1. 中文乱码
2. `articleSectionRef` 命名和模板里的 `ref` 要统一
3. 首屏内部类名有重复嵌套，结构不够清晰
4. 摘要最好做去 HTML 处理
5. About 路由跳转建议统一成 `/about`

---

## 十二、最小改造顺序

你自己改的时候，按这个顺序最稳:

1. 先把 `Home.vue` 的中文乱码全部替换掉
2. 再把首屏 template 改成视频 Hero 结构
3. 保留文章区逻辑，只调整排版层级
4. 加一个轻量的项目说明区
5. 最后再补视频素材
6. 最后再做样式细化

不要一开始就调动画，不然会很乱。

---

## 十三、当前最推荐的落地版本

如果你想少走弯路，直接做这个版本:

- 首屏: 一个大视频面板
- 视频上覆盖标题、副标题、按钮
- 视频下方: 3 张轻量信息卡
- 第二屏: 搜索 + 分类 + 标签 + 文章列表
- 第三屏: 项目说明

这版最适合你当前项目，也最容易继续往下迭代。

---

## 十四、后续样式交给 Claude 时怎么描述

你后面单独让 Claude 调样式，可以直接这样描述:

`请基于现有 Home.vue 结构，不改接口逻辑，只优化首页视觉。风格参考 Claude 官网，要求简约、克制、暖中性色、大留白、轻毛玻璃、弱阴影、安静的视频首屏，不要做炫技轮播，不要高饱和渐变。`

如果要更具体一点，再补一句:

`首页首屏是项目录屏视频，文案覆盖在视频上层，下面接文章列表和项目说明区，整体重点是内容感和可阅读性。`

---

## 十五、结论

你现在最适合的首页不是“炫幻灯片”，而是:

`简约视频首屏 + 正常博客内容流`

这样既有设计感，也不会牺牲博客本身的可用性。

先把结构搭起来，样式后面再精修，这个顺序是对的。
