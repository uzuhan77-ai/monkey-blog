## 第九步：配置 Gunicorn

创建 systemd 服务文件：

```bash
sudo nano /etc/systemd/system/lyhblog.service
```

填入：

```ini
[Unit]
Description=lyhblog gunicorn service
After=network.target

[Service]
User=www-data
Group=www-data
WorkingDirectory=/var/www/lyhblog/backend
EnvironmentFile=/var/www/lyhblog/backend/.env
ExecStart=/var/www/lyhblog/backend/venv/bin/gunicorn config.wsgi:application --workers 3 --bind 127.0.0.1:8000
Restart=always

[Install]
WantedBy=multi-user.target
```

启动服务：

```bash
# 修改权限
sudo chown -R www-data:www-data /var/www/lyhblog

# 启动 Gunicorn
sudo systemctl daemon-reload
sudo systemctl start lyhblog
sudo systemctl enable lyhblog
sudo systemctl status lyhblog
```

查看日志：

```bash
sudo journalctl -u lyhblog -f
```

## 第十二步：配置 Nginx

在服务器上创建配置文件：

```bash
sudo nano /etc/nginx/sites-available/lyhblog
```

填入（**修改 server_name**）：

```nginx
server {
    listen 80;
    server_name 你的服务器IP 你的域名;

    root /var/www/lyhblog/frontend/dist;
    index index.html;

    # 前端路由
    location / {
        try_files $uri $uri/ /index.html;
    }

    # 后端 API
    location /api/ {
        proxy_pass http://127.0.0.1:8000/api/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    # 静态文件
    location /static/ {
        alias /var/www/lyhblog/backend/staticfiles/;
    }

    # 管理后台
    location /admin/ {
        proxy_pass http://127.0.0.1:8000/admin/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

启用配置：

```bash
# 创建软链接
sudo ln -s /etc/nginx/sites-available/lyhblog /etc/nginx/sites-enabled/

# 删除默认配置（如果冲突）
sudo rm /etc/nginx/sites-enabled/default

# 测试配置
sudo nginx -t

# 重启 Nginx
sudo systemctl restart nginx
sudo systemctl status nginx
```

## 第十四步：配置域名（可选）

### 14.1 域名解析

去你的域名服务商（阿里云万网）：

1. 进入 **域名控制台**
2. 找到你的域名，点击 **解析**
3. 添加 A 记录：

| 记录类型 | 主机记录 | 记录值 |
|---------|---------|--------|
| A | @ | 你的服务器IP |
| A | www | 你的服务器IP |

等待 5-10 分钟生效。

### 14.2 测试域名

```bash
ping 你的域名
```

如果返回你的服务器 IP，说明解析成功。

### 14.3 更新配置

修改后端 `.env`：

```env
DJANGO_ALLOWED_HOSTS=你的服务器IP,你的域名,www.你的域名
CORS_ALLOWED_ORIGINS=http://你的域名,http://www.你的域名
CSRF_TRUSTED_ORIGINS=http://你的域名,http://www.你的域名
```

重启服务：

```bash
sudo systemctl restart lyhblog
```

## 常见问题

### 1. 502 Bad Gateway

检查 Gunicorn 是否运行：

```bash
sudo systemctl status lyhblog
sudo journalctl -u lyhblog -n 50
```

### 2. 静态文件 404

重新收集静态文件：

```bash
cd /var/www/lyhblog/backend
source venv/bin/activate
python manage.py collectstatic --noinput
sudo systemctl restart lyhblog
```

### 3. 前端路由刷新 404

检查 Nginx 配置中的 `try_files` 是否正确。

### 4. CORS 错误

检查后端 `.env` 中的 `CORS_ALLOWED_ORIGINS` 和 `CSRF_TRUSTED_ORIGINS` 是否包含你的域名。
