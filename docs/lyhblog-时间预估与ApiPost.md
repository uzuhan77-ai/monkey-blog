# lyhblog 时间预估与 ApiPost 使用

> 独立文档：写多久能完成、部署要多久、如何用 ApiPost 测接口。

---

## 一、用 ApiPost 测后端

- ApiPost 用法类似 Postman：新建请求，URL 填 `http://127.0.0.1:8000/api/xxx`，看返回状态码和 JSON。
- 前后端分离时建议**先在后端把接口在 ApiPost 里测通**，再对接前端，能少踩坑。
- 不确定是前端还是后端问题时，用 ApiPost 直接调接口，可快速排除后端问题。

---

## 二、整体时间预估（业余做）

| 阶段 | 内容 | 时间（业余） |
|------|------|----------------|
| 环境 + 空项目 | 两个文件夹、Django 项目、Vue3 项目、一条测试 API、前端能请求到 | **0.5～1 天** |
| 核心功能 | 文章 CRUD、分类/标签、列表+详情页、简单后台（发/改文章） | **3～5 天** |
| 评论 + 用户 | 评论接口+页面、用户注册/登录、token、权限 | **2～3 天** |
| 锦上添花 | 友链、关于页、简单统计、样式收拾 | **1～2 天** |
| 部署 | 服务器/域名、Nginx、后端 run、前端 build 部署、HTTPS（可选） | **1～2 天** |

**合计：大约 1～2 周**（每天能投入 2～3 小时）。  
全职做、或只做「文章+分类+评论+登录」最小闭环，可压到 **5～7 天**。

---

## 三、影响进度的因素

- **熟不熟 Django / Vue**：熟的话每个阶段能省不少时间。
- **做多少功能**：只做文章+评论+登录会比 POEMON 少很多模块，会快不少。
- **是否第一次部署**：第一次搞 Nginx + 域名会多花半天到一天。
- **用 ApiPost**：先把接口在 ApiPost 里测通再对接前端，能少踩「不知道是前端还是后端问题」的坑。

---

## 四、建议节奏（配合 ApiPost）

1. **后端先跑起来** → 在 ApiPost 里把「文章列表、文章详情、发文章」等接口都测通（状态码、返回 JSON 都符合预期）。
2. **再写前端页面** → 用 axios 调这些已测过的接口，只关心页面和交互。
3. **每做完一块就本地多点点** → 避免最后一起联调。

整体 **1～2 周内写完并部署好** 是合理预期；想稳一点就按 2 周排期。
