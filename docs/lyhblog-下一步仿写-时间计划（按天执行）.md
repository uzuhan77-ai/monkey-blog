## 0. 你现在的进度（仓库真实代码核对）

### 已完成（可用）

- **后端**（`lyhblog-backend`）
  - 文章列表（分页+可按分类筛）：`POST /api/article/listArticle/`（`current/size/category_id` → `data/total`）
  - 文章详情：`GET /api/article/getArticleById/?id=xxx`（→ `data`）
  - 登录/注册：`POST /api/user/login/`（Token）、`POST /api/user/register/`
  - 分类列表：`GET /api/category/listCategory/`
  - 模型：`Article` 关联 `Category`、`Tag`，序列化器已嵌套返回 `category/tags`

- **前端**（`lyhblog-frontend`）
  - 页面：`Home.vue`、`ArticleDetail.vue`、`login.vue`（以及 `register.vue` 路由已预留）
  - 路由：`src/router/index.js`
  - API：`src/api/article.js` + `src/utils/constant.js`

### 当前必须先修的卡点（不修会影响后续所有联调）

- **路由拼写错误**：详情页路由是 `'/articel'`，但首页跳转写的是 `'/article'`
- **列表跳详情代码错误**：`goToDetail` 里 `id` 未定义、形参命名不一致
- **详情页取参不匹配**：详情页用 `route.params.id`，但跳转更像走 `query.id`
- **详情页正文未渲染**：模板正文区域为空
- **详情接口调用方式不匹配**：`ApiArticleDetail` 期望 `params` 对象，但传了单个 id

## 1. “写完”的定义（MVP + 可交付）

### 前台 MVP

- 文章列表/详情（含分页）
- 分类筛选、标签展示
- 评论列表 + 发表评论（登录后）
- 登录/注册 + redirect 回跳

### 后台 MVP

- 文章 CRUD（含分类/标签绑定）
- 评论管理（删除/屏蔽）
- 最小权限控制：普通用户 vs 管理员（后台接口只允许管理员）

### 工程化/可部署

- axios 统一封装（baseURL、拦截器、自动带 token、统一错误处理）
- 路由守卫（requiresAuth）
- 部署文档：前端 build、后端生产配置思路
