### Day 4：AI 助手稳定性优化

**目标**：让 AI 功能从"能用"变成"稳定"

**要做的事**：

2. 优化错误提示

     - `AI 网关连接异常，请检查配置`

3. 增强 JSON 解析稳定性
   - 清理 markdown 代码块包裹

**验收标准**：
- AI 助手连续成功 3 次
- 错误提示用户能看懂
- 不再出现 502 错误

### Day 12：写简历项目描述

**目标**：把项目包装成简历语言

**简历描述模板**：

**项目名称**：
```
前后端分离内容管理系统 + AI 写作助手
Django REST Framework + Vue3 + MySQL
```

**项目描述**：
```
基于 Django REST Framework + Vue3 实现前后端分离内容管理系统，支持文章发布、分类标签管理、评论管理、登录注册与后台权限控制，并集成 AI 写作助手用于自动生成标题、摘要、标签、提纲和 Markdown 草稿。
```

**项目亮点**（4 条）：
1. 基于 Django REST Framework + Vue3 构建前后端分离内容管理系统，完成文章发布、分类标签管理、评论管理、登录注册与后台权限控制等核心功能
2. 封装 Axios 请求层与 Token 鉴权链路，结合 Vue Router 与 Pinia 实现后台访问控制及前后端联调
3. 在后台文章编辑页接入 AI 写作助手，支持根据主题自动生成标题、摘要、标签建议、文章提纲与 Markdown 草稿，并一键回填表单
4. 通过环境变量管理、Nginx + Gunicorn 部署链路与 Linux 环境演练，完善项目从开发到部署的基本闭环

**验收标准**：
- 简历描述可直接使用
