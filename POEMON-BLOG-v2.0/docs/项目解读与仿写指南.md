## 一、项目用到的技术栈

本项目**本身就是 Django + Vue3** 前后端分离架构。

| 层级 | 技术 | 说明 |
|------|------|------|
| **前端** | Vue 3 | Composition API |
| | Vite 5 | 构建工具 |
| | Pinia | 状态管理（用户/管理员 token 等） |
| | Vue Router 4 | 路由（含后台 requiresAuth） |
| | Element Plus | UI 组件库 |
| | ECharts | 图表（如访客统计） |
| | APlayer | 音乐播放器 |
| **后端** | Django 4.1.5 | Web 框架 |
| | Django REST Framework | 提供 REST API |
| | MySQL | 数据库 |
| | 七牛云 | 图片/静态资源存储 |

**结论**：完全可以用 **Django + Vue3** 仿造本系统，或直接在本项目基础上删减/扩展。

## 二、值得借鉴的地方

### 1. 前后端分离与目录清晰

- 根目录下 `My-Blog-Vue3/`（前端）、`My-Blog-Serve/`（后端）分离。
- 仿写时保持「一个前端项目 + 一个 Django 项目」即可。

### 2. API 设计

- 接口统一在 `/api/` 下，按业务分模块：`api/article/`、`api/comment/`、`api/weiYan/`、`api/family/`、`api/user/` 等。
- 后台管理接口带 `admin/`，如 `api/admin/article/`、`api/admin/user/`。
- 仿写时可沿用「模块化 + 前后台区分」的命名方式。

### 3. 认证与权限

- 登录后前端用 Pinia 存 token，请求时带在 Header。
- 后台路由通过 `meta: { requiresAuth: true }` 做守卫，未登录跳转 `/verifyLogin`。
- 后端按 token 校验，区分普通用户与管理员。
- 仿写时可直接复用「token + 路由守卫」这一套思路。

### 4. 配置集中

- 前端：`src/utils/constant.js` 里集中写 `baseURL`、`webURL`、图床、视频等。
- 后端：`luntan/settings.py` 里数据库、SECRET_KEY、邮箱、AI 摘要等。
- 仿写时也建议「一个常量文件 + 一个 settings」，方便部署和环境切换。

### 5. 功能模块划分

- 文章、分类/标签、评论、微言（说说）、树洞、友链、相册/资源、音乐、访客统计、后台管理等。
- 每个模块对应：
  - 后端：`user/views/xxx/` + `appone/models/` 里对应模型
  - 前端：`src/views/` 页面 + `src/api/` 里对应接口封装
- 仿写时可按「一个功能 = 一个后端子模块 + 一个前端 api + 若干页面」来拆。

### 6. 部署方式

- 后端：uWSGI + Nginx；前端：`npm run build` 后把 `dist` 用 Nginx 做静态站，并配置 history 模式。
- 可直接参考 README 中的 Nginx 配置。

## 三、如何阅读这个系统（推荐路线）

按「先整体再单点、再跟一条完整链路」的方式看，会比较容易上手。

### 1. 先看入口与边界

- **后端入口**：`luntan/urls.py`  
  - 把请求分到 `appone`（登录/注册/忘记密码）和 `user`（业务 API）。
- **前端入口**：  
  - `src/router/index.js`：所有页面与后台路由、守卫逻辑。  
  - `src/utils/constant.js`：API 地址、图床等。  
  - `src/utils/request.js`：axios 封装、token、loading。

这样能快速知道「请求从哪进、前端调哪些地址、后端分哪几块」。

### 2. 后端：按「URL → 视图 → 模型」看

- **路由**：`user/urls/index.py`  
  这里定义了所有 API 的 path 和对应的 View（如 `GetArtView`、`ListView`、`WeiYanView` 等）。看这个文件就能知道「有哪些接口、分别叫什么名字」。
- **视图**：`user/views/`  
  按业务分子目录：`article/`、`comment/`、`weiyan/`、`family/`、`client/` 等。每个接口的权限、参数、返回都在对应 view 里。
- **模型**：`appone/models/`  
  如 `article.py`、`comment.py`、`wei_yan.py`、`client.py`、`family.py` 等。了解「存什么数据」时直接看这些 model。

建议先选一个你最关心的功能（例如「文章列表 + 文章详情」），然后：

1. 在 `user/urls/index.py` 里找到 `api/article/...` 的 path；  
2. 打开对应 `user/views/article/xxx.py` 看逻辑；  
3. 在 `appone/models/article.py` 里看字段和关系。  

这样就把「一条请求 → 一个接口 → 一块数据」串起来了。

### 3. 前端：按「页面 → API → 组件」看

- **路由**：`src/router/index.js`  
  可以看到前台（首页、分类、标签、文章、微言、友链、留言、关于等）和后台（文章列表/编辑、用户、评论、树洞、资源、违禁词等）分别对应到 `src/views/` 下的哪些 `.vue`。
- **接口封装**：`src/api/`  
  `article.js`、`comment.js`、`user.js`、`weiYan.js`、`family.js` 等，和后台的模块一一对应。看某个页面时，先看它调了哪些 `api/xxx` 的方法，再在对应 `api/xxx.js` 里看请求的 URL 和参数。
- **页面与组件**：`src/views/`  
  前台页面如 `home.vue`（布局）、`index.vue`、`article.vue`、`weiYan.vue` 等；后台在 `admin/` 下（如 `postList.vue`、`postEdit.vue`、`userList.vue`）；公共组件在 `views/common/`（如评论、播放器、头部等）。

阅读顺序可以是：  
**路由里某个 path → 打开对应 .vue → 看调了哪些 api → 去 api/xxx.js 看请求 → 再到后端 url → view → model。**

### 4. 建议跟一条完整链路：文章

1. **后端**  
   - `user/urls/index.py` 里搜 `article`，得到如 `listArticle`、`getArticleById`、`saveArticle` 等。  
   - 打开 `user/views/article/list.py`、`get.py`、`saveart.py` 看如何查库、返回 JSON。  
   - 看 `appone/models/article.py` 的字段。
2. **前端**  
   - 打开 `src/views/article.vue` 或文章列表所在页面。  
   - 看它调用的 `article.js` 里的方法（如 `listArticle`、`getArticleById`）。  
   - 对照 `constant.js` 里的 `baseURL`，就能和后台 path 对上。

把「文章」这一条链路走通后，评论、微言、友链等都是同一套模式，只是换一个模块目录。

## 四、用 Django + Vue3 仿写时可以怎么做

- **方案 A：在本项目基础上改**  
  保留 Django + Vue3 + DRF + 现有 API 设计，只改 UI、删减或增加功能（例如去掉音乐、树洞，或加上你想要的模块）。数据库可以沿用现有模型，用迁移慢慢改。

- **方案 B：从零仿造**  
  新建一个 Django 项目 + 一个 Vite+Vue3 项目。按「文章 → 分类/标签 → 评论 → 用户/登录」的顺序，先做最小闭环（例如：能发文章、能列表和详情、能评论、能登录后台）。再参考本项目的 `user/views/`、`appone/models/` 和前端的 `api/`、`views/` 设计，把你想做的功能一个个加进去。

## 五、项目结构速查

### 后端 (My-Blog-Serve)

```
luntan/           # 项目配置
  urls.py         # 总路由 → appone + user
  settings.py     # 数据库、密钥、邮箱、CORS 等

appone/           # 登录注册等
  urls/index.py   # api/appone/login, registration, 忘记密码
  models/         # 与 user 共用，见下方
  views/          # login, registration, regret, test

user/             # 业务 API
  urls/index.py   # 所有 api/article、api/comment、api/weiYan 等
  views/          # article, comment, weiyan, family, client, map, resource...

appone/models/    # 数据模型（被 user 使用）
  article.py, comment.py, client.py, wei_yan.py, family.py, sort.py, label.py...
```

### 前端 (My-Blog-Vue3)

```
src/
  router/index.js     # 路由与守卫
  utils/
    constant.js       # baseURL、图床、视频、userId、about 等
    request.js       # axios、token、loading
  api/                # 按模块封装接口
    article.js, comment.js, user.js, weiYan.js, family.js, webInfo.js...
  views/              # 页面
    home.vue, index.vue, article.vue, weiYan.vue, friend.vue...
    admin/            # 后台
    common/           # 公共组件
```
